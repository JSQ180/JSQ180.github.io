阻止事件冒泡

这个是出自 room-item 组件。
问题是：当点击按钮切换图片时，由于图片本身绑定的有跳转功能，所以会在切换完成后立马跳转页面
解决方法：e.stopPropagation()即可

公共头部组件（不同页面不同效果）

分析： header 头部组件原本是放在 App.js 中的，这样，头部组件之下的才是各个页面之间的组件。
但是这样写，header 组件就与当前展示的页面时哪个页面了。
解决方法一： 最简单的方法，就是把 header 组件在每个页面中引入一次。
解决方法二： 在 App.js 中根据 header 组件在 redux 中的不同状态，来控制 不同页面的不同展示效果（采用）

header 状态 redux 存储

页面加载时 dispatch 修改状态

AppHeader 组件响应数据的改变

根据以上思路，当我们进入不同页面时，为 header 组件创建的 redux 的状态都会发生变化，相应的，header 组件应该根据这些状态呈现不同的样式效果。

首先需要拿到当前最新的状态，根据状态来设置最新的 css 样式

Suspense 异步加载导致状态改变而组件不被监听

原因分析

问题：<AppHeader/>组件是在 App.js 内部的，当每进入一个新的页面时，页面懒加载导致 react 底层的 subscribe 无法监听到 store 中的数据改变。

具体情况如图所示：

问题解决

解决方法很简单，将 Suspense 放在 Provider 内部就可以了，证明懒加载也是数据 Provider 的一部分

此时问题就解决了。
但是由于组件异步加载，监听到状态的改变而会多刷新几次。

造成的结果：

当同步加载时

打印结果：

前两次的打印是懒加载下页面 js 未被加载完的情况下，header 组件获取了未 dispatch 的状态，即（老的状态）。当 js 加载完毕，dispatch 后，header 组件监听到状态发生改变，就会再次执行一遍代码
解决方法：
Suspense 不要放在最外部的 index.js 中，放在 App.jsx 中就可以。

因为 routes 中的内容才是我们真正想要异步加载的。
