React项目(爱彼迎)-项目初始化

发表于2022-10-05|更新于2022-10-05|React
|字数总计:3.2k|阅读时长:13分钟|阅读量:

针对所学的 React 知识，对其进行一个项目练习，本项目参考于 coderwhy 的爱彼迎项目进行的书写。

项目规范

  1. 文件夹：文件名称统一小写、多个单词以连接符 (-) 连接
  2. JavaScript 变量名采用小驼峰标识，敞亮全部使用大写字母、组件采用大驼峰
  3. CSS 采用普通 CSS 和 styled-component 结合来编写（全局采用普通 CSS、局部采用 styled-component）
  4. 整个项目统一使用函数式组件，使用 hooks API。
  5. 所有的函数是组建，为了避免不必要的渲染，全部使用 memo 进行包裹
  6. 组件内部的状态，使用 useState、useReducer；业务数据全部放在 redux 中管理
  7. 函数组件内部基本按照如下编写顺序编写代码：
    • 组件内部 state 管理
    • redux 的 hooks 代码
    • 其它 hooks 相关代码（比如自定义 hooks）
    • 其它逻辑代码
    • 返回 JSX 代码
  8. redux 代码规范如下：
    • redux 中 react-redux 和 redux 都会使用
    • 每个模块有自己独立的 reducer 或者 slice，之后合并在一起
    • redux 中会存在共享的状态，从服务器获取到的数据状态
  9. 网络请求采用 axios
    • 对 axios 进行二次封装。
    • 所有的模块请求都会放到一个文件夹中单独管理
  10. 项目使用 AntDesign、MUI
    • 两个 UI 组件库都会使用
    • 多部分组件还是自己进行编写、封装、实现

配置项目别名

原因：当我们的目录结构比较深的情况下，需要 ‘../../../‘ 类似的
而配置好别名之后就可以直接通过 @ 访问 src 目录（大多数都这样配）
这种配置是在 webpack 中配置的，而 react 脚手架 默认是把 webpack 隐藏的

解决方法一： npm run eject // 导出所有的配置，并进行修改。不足之处是如果修改出错，会导致整个项目无法运行（不建议）

解决方法二： craco，即 create-react-app config 缩写。他的使用是将新的配置和原有的 config 配置进行合并

这里也推荐使用 antdesign 官网，也有相关配置：

https://ant-design.gitee.io/docs/react/use-with-create-react-app-cn
